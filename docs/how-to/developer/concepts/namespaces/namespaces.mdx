This guide explores working with namespaces. Namespaces allow users to group applications and resources together. For more information see the [namespaces reference](../../../../reference/configuration/namespaces.md) page.

## Working with namespaces

<Tabs>
<TabItem value="ui" label="UI" default>

### View Namespaces
To view a table of all the namespaces available, navigate via the side menu to the namespaces page.
<img
  src={require('./namespaces-screen.png').default}
  alt="View of the Namespaces screen"
/>

### Create Namespace
Creating a namespace is simple. Click the Create button on the namespaces page to open up the creation modal. Input the namespace name, and then click the Create button and the newly created namespace will appear in the table.
<img
  src={require('./namespaces-create-modal.png').default}
  alt="Create Namespace Modal"
  width="400"
/>

### Delete Namespace
To delete a namespace, click on the action menu in the table row for the namespace you would like to remove. Clicking on the Delete option will open up the modal, where you will be required to confirm the action by entering the namespace name.
<img
  src={require('./namespaces-delete-modal.png').default}
  alt="Delete Namespace Modal"
  width="400"
/>

### Using Namespaces
When creating resources (Applications, Services, etc.) in Epinio you will be prompted to select a namespace for that resource to be associated with. When viewing the lists of resources, you can filter based on the namespace using the dropdown menu in the top bar.
<img
  src={require('./namespaces-dropdown.png').default}
  alt="Namespace Filter Dropdown"
/>

<TabItem value="cli" label="CLI">

Once Epinio is deployed, check the existence of the pre-configured namespace `workspace`:

```console
epinio namespace list
```

Output:

```
🚢  Listing namespaces

✔️  Epinio Namespaces:
|   NAME    |            CREATED            | APPLICATIONS | CONFIGURATIONS |
|-----------|-------------------------------|--------------|----------------|
| workspace | 2023-03-14 09:48:11 +0100 CET |              |                |
```

Create a new namespace:

```console
epinio namespace create newnamespace
```

Output:

```
🚢  Creating namespace...
Name: newnamespace

✔️  Namespace created.
```

Inspect the details of a namespace (creation date, applications, configurations):

```console
epinio namespace show newnamespace
```

Output:

```
🚢  Showing namespace...
Name: newnamespace

✔️  Details:
|      KEY       |             VALUE             |
|----------------|-------------------------------|
| Name           | newnamespace                  |
| Created        | 2023-03-14 15:27:48 +0100 CET |
| Applications   |                               |
| Configurations |                               |
```

Epinio creates the default namespace `workspace` at installation time, and `epinio login` sets it as the current namespace. To see the current namespace, invoke the command without an argument:

```console
epinio target
```

Output:

```
✔️
Currently targeted namespace: workspace
```

To target the namespace `newnamespace` for future Epinio operations:

```console
epinio target newnamespace
```

Output:

```
🚢  Targeting namespace...
Name: newnamespace

✔️  Namespace targeted.
```

To start working with a different existing namespace, run `epinio target` again with its name. Finally, delete the namespace `newnamespace`:

```console
epinio namespace delete newnamespace
```

Output:

```
You are about to delete namespace newnamespace and everything it includes, i.e. applications, configurations, etc. Are you sure? (y/n): y

🚢  Deleting namespace...
Name: newnamespace

✔️  Namespace deleted.
```
